Case study

Nawras AI Platform

A low-code platform for building, grounding, and shipping multi-step agentic AI workflows in production.

Role AI Developer & Automation Engineer Org Al-Obeikan Digital Solutions Years 2023 to present
Get in touch → All work →

Most teams can prototype an agent. Far fewer can run one reliably against real enterprise data.

Nawras is the platform I built to close that gap. It gives teams a visual canvas to compose multi-step agentic workflows, route them across multiple LLM providers behind one interface, and ground every response in their own data through RAG and Graph RAG.

Guardrails, evaluation, memory, a tools builder, MCP connectivity, and end-to-end tracing are built in. A workflow that works in a demo keeps working in production, where it is observable and safe to change.

Capabilities

Everything an agent needs to leave the demo.

Visual workflow builder

A drag-and-drop canvas to compose, branch, and version multi-step agent workflows without writing glue code.

Multi-provider routing

One interface in front of many LLM providers, with per-step model choice and fallback.

RAG and Graph RAG

Grounding over embeddings, vector search, and a knowledge graph for accurate, explainable answers.

Guardrails

Input and output checks that keep responses safe and on policy.

Evaluation

RAG-based scoring that catches quality regressions before they ship.

Memory and MCP

Persistent context plus Model Context Protocol connectivity to external tools.

How a workflow runs.

01

Compose

Build the workflow visually: steps, branches, tools, and prompts on one canvas.

02

Route

Each step routes to the chosen LLM provider through a single gateway, with fallback.

03

Ground

Retrieval over embeddings and a knowledge graph injects the right context with RAG and Graph RAG.

04

Guard and evaluate

Guardrails check every input and output; evaluation scores quality continuously.

05

Trace

End-to-end tracing and monitoring make each run observable and debuggable.
